Introduction

    The Visualization Software system is Win95 / Win NT (TM of Microsoft Corp., USA) based. This means, it requires a number of system libraries as well as Windows I/O services and is not executable without Windows.

    However, the Data Base (containing the spectra data and the according information files) is organized as a set of plain ASCII text files to give a maximum of freedom in use of the data base content for other purposes (e.g. for publication with any common text editor, for use with non-Windows software). It is accompanied by a system of HTML substance descriptions, which can be used by any standard Web browser.

    Therefore the data base doesn´t have the close connection to Windows application standards as the executeable visualization software. But, the resulting (almost) unlimited application spectra of the data set requires some additional processing time for translating the plain ASCII text strings into computer-readable numbers. But, this is notizeable only with large data sets.

 Data Base Structure

    The Data Base is structured as shown below:
     
    • Each folder is named as the contained species
    • The substance name is the leading part of the file name
    • Each single measurement is stored as an unique file set
    • The file extension distinguishes between data and information.

File Name Convention

    The file name is the users entrance key to data and information. The well-known older ISO 9660 (8+3) file name definition of DOS are not sufficient to generate unique file names without cryptic encoding of the file names. Therefore the newer LONG FILE NAME (Joliet) method (240 + 3) has been applied.

    The file naming is primarely oriented to the user, the computer takes the file names as they are and needs only the extension to discriminate between data and information.

    A file name is defined as
     

      Substance_Temperature_ID.Ext
    The separator between the file name elements is a „_“ .
    In any case the name of a data file and the according information file has been the same.

    Example:
     

      Folder:     Nitrogen oxides

      Data :      NO2_233K_1.sff
      Info:       NO2_233K_1.inf
                   |   |  |  |
      substance   -|   |  |  |
      temperature -----|  |  |
      meas. Id    --------|  |
      extension   -----------|

Data File Specification

    Each data file (extension is .SFF, which is for Spectra File Format) consist of a describing 6-line header part (each headeris terminated with <cr><lf>):
     
      ; ** substance **
      ; Literature reference
      ; Temperature xxx K
      ; lambda  sigma
      ; nm      cm^-2^
      ;------------------
    followed by the list of (l,s) value pairs in one of the following formats:
     
       l1 s1 <cr><lf>
          or
       l1 s1   l2 s2 <cr><lf>
          or
       l1 s1   l2 s2   l3 s3 <cr><lf>
    1. The list is increasing (l1 < l2) ordered
    2. Each data line starts with (at least) one <blank> character and is terminated by <cr><lf>
    3. up to 3 (l, s) pairs are supported in one line
    4. The separator between the values is (at least) one <blank> character
    5. Data format l ::= ddd.dd[d[d]]
    6. Data format s ::= d.ddd[d[d]]E-[dd]dd

    Example of  a data file
     

      File name:  Br2O_298K_1.sff
     
      ** Br2O **
      ;Orlando and Burkholder ...
      ;Temperature  298 K
      ;lambda       sigma
      ;nm          cm^-2^
      ;------------------
       196.00   1.740E-17
       198.00   1.930E-17
       200.00   2.000E-17
       202.00   1.950E-17
       204.00   1.820E-17
       206.00   1.610E-17
       208.00   1.340E-17
       210.00   1.090E-17
       212.00   8.440E-18
       214.00   6.420E-18
       216.00   4.880E-18
        ::         ::
        ::         ::

Information File Specification

    The information file (extension is .INF) contains  the more detailed description. It can be edited in case the related data is modified.

    Example of an information file
     
    File name:  Br2O_298K_1.inf

      Substance: Br2O
      Orlando, J.J., and J.B. Burkholder

      Gas Phase UV/Visible Absorption Spectra of HOBr and Br2O

      J. Phys. Chem., 99 (1994), 1143-1150.

      Data: table
      Resolution:     2.0 nm
      Temperature:  298.  K
      Temperature dependence: no

      Comment:
      Br2O was synthesized via rection of Br2 with HgO. Br2O could not be obtained free of Br2 contamination. Br2O  absorption cross sections should be treated as an upper limit.

Start and Operating

Auto Start

    As default, after loading the media into the drive the Windows AUTO-RUN system procedure will be activated and the ASPO Launcher (details see on next page) will be loaded and executed.
     

Manual Start

    In case, the default system feature of AUTO-RUN is disabled, the software can be started using the common system procedures:
     
    via the Explorer by selecting and double-clicking ASPO Launcher.exe

    via the Windows START button by EXECUTE <CD>:\ASPO Launcher.exe
     
    As a first action the Intro Splash Screen appears during the period of installing the executeable files to the PC memory. It disappears automatically short time after finishing the installation process. The screen shows the image on the title page.

Operating from Hard Disk

    As default, the software modules and the data base are executeable directly from the CD-ROM.

    In case, the data base has to be extended  or the filtering procedure should be used, the following procedure is recommended:

    -  create a new folder <NEW>on hard disk
    -  switch over to this new folder
    -  copy all files from the CD root directory
    -  copy the FILTERED folder
    -  copy the DBTEXT folder and its content
    -  copy all <SPECIES> folders and all
        required substances
     
    The folder structure (now sub-folders of <NEW>), the folder names and the name of any copied file should not be changed.

    This data base structure is open to be enlarged with additional substances etc.

    It is the main control window.

    • The Substances button opens the hidden buttons to access the substance description and some specific definitions
    • The Graphics button opens the hidden buttons for starting the ASPO Viewer,  the Filter and the Manual (HTML file)
    • The Exit button closes the application and exit to the Windows system.

The LIST EDITOR Window

    The List Editor Window is the input selection window. It is activated by the OPEN LIST EDITOR button and works as a default to the directory structure created on hard disk by the AUTO RUN procedure. Alternatively it can be changed to the data base structure on the CD-ROM.

    Those mouse actions are supported:

    • the left mouse button selects a file set. The header of the data file will be listed in the header display area.
    • the right mouse button opens the Info display window to list the information file of the selected file set

Filter

    The filter converts a spectra data file with much higher resolution and/or individually spacing   into a standardized output file as used for modelling or spectroscopy  purposes.

    The filter program flow  has been adopted from a FORTRAN source, provided by E.P. Roeth (University of Essen).

    The following filter modes are provided:
     

    • Equidistant Mode
    • Modelling Mode
    The intervals definition for the modelling mode is provided on a definition file.

    The equidistant intervals can be set by changing the default parameters  during filtering process.

    Mathematical remarks:
    The used filtering mechanism is a sub-sampling process and requires at least one value  for each output interval. All input data of  an interval are integrated and converted into a single mean value for this interval.

    In case, the lambda region of the input data is smaller than the selected output region, empty  intervals at begin and end of the selected integration region are skipped. The output file starts with the first non-empty interval and ends with the last non-empty interval of the selected region.

    If  the difference of two adjacent  input data is larger than the selected interval spacing, some intervals are skipped too. In this case, the filtering works mathematically correct, but doesn’t make sense from the scientific point of view.

back

Equidistant Filter Mode

    for spectroscopy purposes subsamples the input data into equidistant spaced intervals (as well as default settings and user selectable)

    Default settings are:

       Lower limit     : 200 nm
       Upper limit     : 460 nm
       Interval width :     2 nm (calc.)
       Interval No     : 130
       
    User selectable
       Lower lim. min  :  100  nm
       Lower lim. max :  990 nm
       Upper lim. min  : LL + 10 nm
       Upper lim. max : 1000 nm
       Interval width   : > 0.05 nm  (calc.)
       Interval No       : 50 - 200
 

Modelling Filter Mode

    for modelling purposes subsamples the input data into individual spaced intervals of specific scientific interest (e.g. the spectral region of the Huggins bands)

    Default values are:

      Default settings are:
      Lower limit :  184.65 nm
      Upper limit :  730.00 nm
      Interval width :  individual spaced
      Interval No : 130

       stored on MODEL.PAR

back

Filter Operation

    The filter is activated by the GRAPHICS / FILTER button from the ASPO Launcher window.

    The selection of the input file is similar to the procedure with the LIST EDITOR Window.

     After selection of the filter mode the process of filtering is started by clicking the START button. The process status is indicated on message boxes and with the status indicators.

     The process terminates automatically after storage of the results file.

    The results file name extends the original input file name with
     
            _m-filt               for modelling mode      or
            _exxx-filt          for equidistant mode
    (with xxx is the number of selected  intervals). The file extension remains .sff

    Example:
     
             Input file:      Cl2Br_280K_1.sff
              Results file: Cl2Br_280K_1_e200-filt.sff

    The results file is stored after filtering on the hard disk in the folder FILTERED (as default), or, in case the folder is write-protected (e.g. while working on CD-ROM) or not available, in an user-selectable folder.

    After end of  filtering process the process can be repeated (with other parameter setting or with another input file) or closed by clicking the EXIT button.

    Messages:
     
    File
      The filter is in Modelling mode and uses the parameter setting from MODEL.PAR
       
    Default
      The filter is in Equidistant mode and uses the default  parameters
       
    Specific
      The filter is in Equidistant mode with user-specific parameter setting
       
    Parameter file missing
      Please check the existence of the selected file MODEL.PAR in the working directory
       
    Aborted
      The filter is not able to create an output file: the output folder doesn’t exist or is write protected and/or the user has not made another selection
       
    no data
      The number of input data is too small for filtering; no output file is created
       
    Region
      The region of lambda doesn’t fit with the selected filter interval borders;
      no output file is created
       
    OK
      The filtering was successful and an output file was created on either the default folder FILTERED or an user-selected folder
       
    For troubleshooting purposes only:

    p0/1 ok     Filtering parameters ok

       
    Start         Starting point reached
       
    Run          Interval added to the output file
